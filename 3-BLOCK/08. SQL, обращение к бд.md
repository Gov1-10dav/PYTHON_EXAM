## Базы данных и СУБД

**База данных (БД)** — организованное хранилище данных в виде взаимосвязанных файлов.

**СУБД (Database Management System)** — комплекс программ для создания, ведения и совместного использования БД. Задачи СУБД: хранение, выгрузка, контроль доступа, резервное копирование.

**PostgreSQL** — свободная объектно-реляционная СУБД на основе SQL. Поддерживает ACID, первичные и внешние ключи, индексы, транзакции, широкий набор типов данных (uuid, json, массивы и др.).

---

## ERD и ключи

**ERD (Entity-Relationship Diagram)** — диаграмма связи сущностей — визуализация связей между таблицами.

- **PRIMARY KEY** — уникально идентифицирует запись; не NULL; одна на таблицу (может быть составной).
- **FOREIGN KEY** — связывает дочернюю таблицу с родительской.

**Транзакции (ACID):**
- **Атомарность** — транзакция выполняется целиком или не выполняется.
- **Согласованность** — данные остаются в допустимом состоянии.
- **Изолированность** — транзакции не влияют друг на друга.
- **Стойкость** — результат сохраняется даже при сбое.

## SQL — Structured Query Language

Группы операторов:
- **DDL** (Data Definition Language): `CREATE`, `DROP`, `ALTER`
- **DML** (Data Manipulation Language): `SELECT`, `INSERT`, `UPDATE`, `DELETE`
- **DCL** (Data Control Language): `GRANT`, `REVOKE`
- **TCL** (Transaction Control Language): `BEGIN`, `COMMIT`, `ROLLBACK`

### Основные конструкции

```sql
-- Выборка всех данных
SELECT * FROM members;

-- Фильтрация
SELECT * FROM members
WHERE surname = 'Smith' OR firstname = 'David';

-- Ограничение строк
SELECT "name" FROM facilities
WHERE membercost IN (0, 5, 10)
LIMIT 10;

-- Агрегация и группировка
SELECT surname, COUNT(surname)
FROM members
WHERE recommendedby IS NOT NULL
GROUP BY surname
HAVING COUNT(surname) > 1;

-- Сортировка
SELECT * FROM bookings
ORDER BY starttime DESC;

-- Агрегатные функции: MIN, MAX, SUM, AVG, COUNT

-- Псевдонимы
SELECT COUNT('surname') AS "total_members" FROM members;

-- JOIN — объединение таблиц
SELECT f.name, b.starttime
FROM bookings b
JOIN facilities f ON f.facid = b.facid;

-- Виды JOIN:
-- INNER JOIN  — пересечение (только совпадающие)
-- LEFT JOIN   — все из левой + совпадения из правой (NULL если нет)
-- RIGHT JOIN  — все из правой + совпадения из левой
-- FULL OUTER JOIN — все из обеих (NULL там, где нет совпадения)

-- Несколько JOIN
SELECT f.name, b.starttime, m.firstname, m.surname
FROM bookings b
JOIN facilities f ON f.facid = b.facid
JOIN members m ON b.memid = m.memid
WHERE firstname = 'David' AND DATE(b.starttime) > '2010-01-01';
```

**Порядок выполнения запроса:**
```
SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
```

### Подключение и запрос

```python
import psycopg2

connection = psycopg2.connect(
    database='exercises',
    host='localhost',
    user='postgres',
    password='password'
    # port=5432  # порт по умолчанию
)

cursor = connection.cursor()

cursor.execute("""
    SELECT *
    FROM cd.bookings   -- cd — схема, bookings — таблица
    LIMIT 10
""")

results = cursor.fetchall()   # список кортежей
print(results)

# Обязательно закрывать
cursor.close()
connection.close()
```

### Через pandas (удобнее)

```python
import pandas as pd

conn_uri = "postgresql://postgres:password@localhost/exercises"

# Чтение таблицы
df = pd.read_sql("SELECT * FROM cd.bookings", conn_uri)
df.head()

# Запрос с JOIN и парсингом дат
df_1 = pd.read_sql("""
    SELECT
        b.starttime,
        b.slots,
        m.firstname,
        m.telephone
    FROM cd.bookings b
    INNER JOIN cd.members m ON b.memid = m.memid
""", conn_uri, parse_dates=["starttime"])

df_1.sample(10, random_state=42)

# Запись обратно в БД
df.to_sql(
    "new_table",        # имя таблицы
    conn_uri,
    schema="cd",
    if_exists='replace' # 'fail' | 'replace' | 'append'
)
```

## ORM через SQLAlchemy

**ORM (Object-Relational Mapping)** — управление таблицами через объекты Python.

```python
from sqlalchemy import Column, Integer, String, Boolean, create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

SQLALCHEMY_DATABASE_URL = "postgresql://username:password@localhost/database"

engine = create_engine(SQLALCHEMY_DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True)
    name = Column(String)
    surname = Column(String)
    age = Column(Integer)
    sex = Column(Boolean)

# Создать таблицы
if __name__ == "__main__":
    Base.metadata.create_all(engine)
```

**Типы SQLAlchemy ↔ Python ↔ SQL:**

| SQLAlchemy | Python | SQL |
|-----------|--------|-----|
| Integer | int | INTEGER |
| String | str | TEXT |
| Float | float | FLOAT |
| Boolean | bool | BOOLEAN |
| Date | datetime.date | DATE |

## Краткое сравнение подходов

| Подход | Когда использовать |
