### **1. Виртуальное окружение**
---
#### **Виртуальное окружение Python** позволяет создавать изолированные среды с независимыми наборами пакетов docs.python.org. Это помогает избежать конфликтов зависимостей между проектами.

Когда вы работаете в виртуальном окружении, Python использует пакеты и настройки _только_ из этой активной среды. Это значит, что библиотеки, которые вы устанавливаете для одного проекта (например, через `pip install`), не будут засорять глобальный Python или конфликтовать с версиями библиотек из другого проекта.

---
##### **Создание и использование:**

`#Создание виртуального окружения`
`python -m venv .venv`

`#Активация на Windows`
`.venv\Scripts\activate`

`#Активация на Linux/Mac`
`source .venv/bin/activate`

`#Деактивация`
`deactivate`

Обычное расположение для виртуального окружения — `.venv`, что делает директорию скрытой в терминале docs.python.org.
