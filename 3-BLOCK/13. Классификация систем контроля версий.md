## Классификация систем контроля версий

| Тип VCS | Описание архитектуры | Плюсы | Минусы | Пример |
| --- | --- | --- | --- | --- |
| Локальные | БД с изменениеями лежит только на одном компьтере пользователя | Легко использовать | Не подходит для командной работы, ненадежно (одна точка отказа) | RCS, SCCS |
| Централизированные | БД с изменениями, файлами и историей находятся на одном выделенном сервере | Удобный контроль доступа | Ненадежно (одна точка отказа) | SVN, Perforce |
| Распределенные | Каждый участнмк проекта имеет полноценный локальный репозиторий со всей историей изменений | Автономность, надежность, скорость работы | Иногда требует много места на диске | Git |

## Git

Git - распределенная система контроля версий, созданная Лиунсом Торвальдсом (создателем ЛИНУКС!!!!) в 2005м году для раработки ядра Линукс. Де-факто является стандартом индустрии. 

#### Преимущества и особенности Git:
- Локальные операции - большинство действий не требует подключения к сети
- Дешевые ветки - переключение и слияние веток происходят мгновенно
- Сохранность данных - Git использует хеширование SHA для проверки целостности файлов, что делает изменение истории невозможным

#### Экосистема Git
К сожалению, GitHub является крупнейшей платформой для хостинга кода при помощи Git

#### Использование Git
Епта ну все не дурачки знать должны но пройду по базе
- git init - инициализировать гит в репозитории
- git add ... - отслеживать изменение файлов
- git commit -m "я очень крут" - сделать "слепок" измененных файлов
- git push {url} {ветка} - сохранить последний "слепок" на сервере
