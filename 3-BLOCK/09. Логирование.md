# 9. Логирование в Python (logging)

---

## Что такое логирование?

**Логирование (logging)** — запись событий, возникающих во время работы программы. Помогает отлаживать, мониторить и анализировать поведение приложения.

В Python есть встроенный модуль `logging`.

```python
import logging
```

---

## Уровни логирования

5 уровней по возрастанию серьёзности:

| Уровень | Числовое значение | Назначение |
|---------|------------------|-----------|
| DEBUG | 10 | Отладочные сообщения, диагностика |
| INFO | 20 | Нормальная работа фрагментов кода |
| WARNING | 30 | Предупреждения (по умолчанию!) |
| ERROR | 40 | Ошибки, часть программы не работает |
| CRITICAL | 50 | Критические ошибки, угроза работе приложения |

```python
logging.debug("A DEBUG Message")
logging.info("An INFO")
logging.warning("A WARNING")
logging.error("An ERROR")
logging.critical("A message of CRITICAL severity")
```

По умолчанию в консоль выводятся только WARNING и выше:
```
WARNING:root:A WARNING
ERROR:root:An ERROR
CRITICAL:root:A message of CRITICAL severity
```

---

## Базовая настройка — `basicConfig()`

### Логирование в консоль с нужным уровнем

```python
logging.basicConfig(level=logging.DEBUG)
logging.debug("Теперь debug тоже видно")
```

### Логирование в файл

```python
logging.basicConfig(
    level=logging.INFO,
    filename="py_log.log",
    filemode="w"   # "w" — перезаписывать; "a" — дописывать (default)
)
```

Параметры `basicConfig()`:
- `level` — минимальный уровень для записи.
- `filename` — имя файла журнала.
- `filemode` — режим работы с файлом (`"w"` или `"a"`).
- `format` — формат строки лога.

> ⚠️ `basicConfig()` работает только при первом вызове. Повторные вызовы игнорируются (если не передать `force=True`).

---

## Форматирование сообщений

```python
logging.basicConfig(
    level=logging.INFO,
    filename="py_log.log",
    filemode="w",
    format="%(asctime)s %(levelname)s %(message)s"
)
```

Доступные атрибуты формата:

| Атрибут | Что выводит |
|---------|-------------|
| `%(asctime)s` | Дата и время |
| `%(levelname)s` | Уровень логирования |
| `%(message)s` | Текст сообщения |
| `%(name)s` | Имя логгера |
| `%(filename)s` | Имя файла с кодом |
| `%(lineno)d` | Номер строки |

## Логирование переменных и исключений

```python
x, y = 3, 4
logging.info(f"x={x}, y={y}")

try:
    result = x / y
    logging.info(f"x/y = {result}")
except ZeroDivisionError:
    logging.error("ZeroDivisionError", exc_info=True)  # включает traceback
```

Альтернатива — `logging.exception()` (автоматически добавляет traceback):

```python
x_vals = [2, 3, 6, 4, 10]
y_vals = [5, 7, 12, 0, 1]

for x, y in zip(x_vals, y_vals):
    try:
        logging.info(f"Вычисляю {x}/{y}")
        logging.info(f"Результат: {x/y}")
    except ZeroDivisionError:
        logging.exception("Деление на ноль!")
```

## Пользовательские логгеры

Для многомодульных приложений лучше создавать отдельные логгеры:

```python
# module_a.py
import logging

logger = logging.getLogger(__name__)  # имя = имя модуля

def some_function():
    logger.debug("Отладочное сообщение из module_a")
    logger.warning("Предупреждение")
```

```python
# main.py
import logging

logging.basicConfig(
    level=logging.DEBUG,
    format="%(asctime)s [%(name)s] %(levelname)s: %(message)s"
)

import module_a
module_a.some_function()
```

## Обработчики (Handlers)

Позволяют отправлять логи в разные места одновременно.

```python
import logging

logger = logging.getLogger("my_app")
logger.setLevel(logging.DEBUG)

# Обработчик для файла
file_handler = logging.FileHandler("app.log")
file_handler.setLevel(logging.WARNING)

# Обработчик для консоли
console_handler = logging.StreamHandler()
console_handler.setLevel(logging.DEBUG)

# Формат
formatter = logging.Formatter("%(asctime)s - %(levelname)s - %(message)s")
file_handler.setFormatter(formatter)
console_handler.setFormatter(formatter)

logger.addHandler(file_handler)
logger.addHandler(console_handler)

logger.debug("debug — только в консоль")
logger.warning("warning — и в консоль, и в файл")
```

Основные типы обработчиков:
- `StreamHandler` — вывод в консоль (stderr).
- `FileHandler` — запись в файл.
- `RotatingFileHandler` — ротация файлов по размеру.
- `TimedRotatingFileHandler` — ротация по времени.
- `SMTPHandler` — отправка по email.

## Краткий итог

```python
# Быстрый старт — всё что нужно для простого логирования
import logging

logging.basicConfig(
    level=logging.INFO,
    filename="app.log",
    filemode="a",
    format="%(asctime)s %(levelname)s %(message)s"
)

logging.info("Приложение запущено")
logging.warning("Что-то пошло не так")
logging.error("Ошибка!", exc_info=True)
```

| Концепция | Суть |
|-----------|-----|
| Уровни | DEBUG < INFO < WARNING < ERROR < CRITICAL |
| `basicConfig()` | Глобальная настройка root-логгера |
| `format` | Кастомизация строки лога |
| `exc_info=True` | Добавление traceback к сообщению |
| Handler | Направление логов (файл, консоль, email) |
| `getLogger(__name__)` | Именованный логгер для модуля |
