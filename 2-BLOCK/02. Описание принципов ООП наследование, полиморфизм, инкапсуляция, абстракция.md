### **2. Описание принципов ООП: наследование, полиморфизм, инкапсуляция, абстракция**
---
#### **К основным принципам ООП относятся:** инкапсуляция, наследование, полиморфизм и абстракция.

---
#### 1. Инкапсуляция (Encapsulation)

**Инкапсуляция** позволяет скрывать внутренние детали реализации объекта и работать только через его внешний интерфейс. 
В Python это достигается публичными, защищёнными и приватными атрибутами (`name`, `_name`, `__name`).

`class User:`
    `def __init__(self, login, password):`
        `self.login = login       # Публичный`
        `self.__password = password  # Приватный (сокрыт)`

    ``def check_password(self, pwd): # Публичный метод (интерфейс)``
        ``return self.__password == pwd``

`u = User("admin", "12345")`
`print(u.__password)  # Ошибка! Доступ запрещен`
`print(u.check_password("12345")) #True. Работаем через интерфейс.`

---
#### 2. Наследование (Inheritance)

**Наследование** позволяет создать **новый класс** на основе уже существующего (родительского). Новый класс (потомок) перенимает все атрибуты и методы родителя, но может добавлять свои или переопределять старые.

В Python поддерживается **множественное наследование** (один класс может иметь несколько родителей). Для вызова методов родителя используется функция `super()`.

`class Animal:          # Родительский класс`
    `def breathe(self):`
        `print("Дышит воздухом")`

`class Dog(Animal):     # Дочерний класс (наследует Animal)`
    `def bark(self):    # Собственный метод`
        `print("Гав!")`

`dog = Dog()`
`dog.breathe()  # Метод унаследован от Animal`
`dog.bark()     # Собственный метод`

---
#### 3. Полиморфизм (Polymorphism)

Способность **разных** классов реагировать на **один и тот же** вызов метода по-своему. То есть у разных объектов может быть метод с одинаковым именем, но реализующий разную логику.

Python не требует, чтобы объекты имели общего родителя для полиморфизма. Главное чтобы у них был метод с одинаковым именем.

`class Cat:`
    `def voice(self): print("Мяу!")`

`class Dog:`
    `def voice(self): print("Гав!")`
    
`# Полиморфизм в действии: мы не знаем, кто перед нами,`
`# но знаем, что у них у всех есть метод .voice()`
`animals = [Cat(), Dog()]`
`for animal in animals:`
    `animal.voice()  # Каждый объект реагирует по-своему`

---
#### 4. Абстракция (Abstraction)

**Абстракция** заключается в том, чтобы выделять в объекте только **важные характеристики** и скрывать **ненужные** детали реализации (предоставляет удобный интерфейс для работы, а внутренние механизмы остаются скрытыми).

Реализуется через встроенный модуль `abc` (Abstract Base Classes). Нельзя создать экземпляр абстрактного класса — можно только унаследоваться от него и реализовать все абстрактные методы

`from abc import ABC, abstractmethod`

`# Абстрактный класс (шаблон)`
`class Figure(ABC):`
    `@abstractmethod`
    `def area(self):`
        `pass  # Нет реализации! Только контракт.`

`# Конкретный класс (реализация)`
`class Rectangle(Figure):`
    `def __init__(self, a, b):`
        `self.a, self.b = a, b`
        
    `def area(self):  # Обязательно реализуем, иначе будет ошибка`
        `return self.a * self.b`

`# fig = Figure()  # Ошибка! Нельзя создать объект абстрактного класса`
`rect = Rectangle(2, 3)`
`print(rect.area())  # 6`
