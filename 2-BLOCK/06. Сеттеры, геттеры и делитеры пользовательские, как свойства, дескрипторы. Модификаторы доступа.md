### 6. Сеттеры, геттеры и делитеры: пользовательские, как свойства, дескрипторы. Модификаторы доступа
---
#### Пользовательские геттеры и сеттеры
**Геттер** — метод для получения значения атрибута. 
**Сеттер** — метод изменения значения атрибута.
Используются для сокрытия внутренней реализации и добавления логики при доступе:
```
class Label:
    def __init__(self, text, font):
        self._text = text  # непубличный атрибут

    def get_text(self):
        return self._text

    def set_text(self, value):
        self._text = value.upper()  # дополнительная логика
```
Геттеры и сеттеры позволяют скрыть внутреннюю реализацию класса и при необходимости добавить проверки при чтении или изменении данных.

---
#### Свойства через `@property`
В Python вместо явного вызова геттеров и сеттеров используют встроенный декоратор `@property`. Он позволяет обращаться к методу как к обычному атрибуту.
```
class Employee:
    def __init__(self, name, birth_date):
        self.name = name        # вызовет setter!
        self.birth_date = birth_date

    @property
    def name(self):             # геттер
        return self._name

    @name.setter
    def name(self, value):      # сеттер
        self._name = value.upper()

    @name.deleter
    def name(self):             # делитер
        del self._name
```
Обращение к `emp.name` вызывает геттер, присваивание `emp.name = "John"` — сеттер, `del emp.name` — делитер. Пользователь класса не замечает разницы с обычным атрибутом.

---
#### Дескрипторы
**Дескриптор** — объект, который управляет доступом к атрибуту класса.
Для этого реализуются специальные методы:
- `__get__`
- `__set__`
- `__delete__`
Используются, когда нужно применить одну и ту же логику к **нескольким атрибутам** без дублирования кода.

---
#### Модификаторы доступа
В Python нет строгих модификаторов доступа, но существует соглашение.
Использование непубличных атрибутов является частью **инкапсуляции — сокрытия внутренней реализации класса.**
| Запись | Уровень доступа | Поведение |
| ---| ------| -------|
| `self.name` | **Публичный** | Доступен везде | 
| `self._name` |  **Защищённый** | Соглашение: не трогать снаружи; доступен в подклассах | 
| `self.__name` | **Приватный** | Python применяет name mangling: атрибут хранится как `_ClassName__name` |
