## 14. Парадигмы программирования: сущность, сходства, отличия, примеры

**Парадигма** — стиль и подход к организации и написанию кода. Делятся на **императивные** (как сделать) и **декларативные** (что сделать).

| **Парадигма**           | **Суть подхода**                                                                        | **Ключевые понятия**                                                     | **Примеры** |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- | ------------------------ |
| **Процедурная**       | Программа как последовательность команд.                             | Переменные, функции, циклы.                                             | C, чистый Pascal   |
| **ООП**                       | Программа как сеть взаимодействующих объектов.                  | Классы, инкапсуляция, наследование, полиморфизм.     | Java, C++, Python        |
| **Функциональная** | Программа как вычисление чистых математических функций. | Неизменяемость, функции высшего порядка, рекурсия. | Haskell, Lisp, Ellang    |

* **Сходства:** Тьюринг-полнота (любую задачу можно решить на любой парадигме), общая цель — управление сложностью кода. Современные языки (включая Python) —  **мультипарадигменные** .
* **Отличия:** Императивный код/ООП постоянно изменяет состояние данных (мутабельность) и использует циклы. Функциональный код запрещает изменять данные (иммутабельность) и использует рекурсию или методы вроде `map` / `filter`.

```Python
# Императивный стиль (цикл и изменение переменной)
res = 0
for x in [1, 2, 3, 4]: 
    if x % 2 == 0: res += x**2

# Функциональный стиль (декларативно, без циклов)
res = sum(map(lambda x: x**2, filter(lambda x: x % 2 == 0, [1, 2, 3, 4])))
```
