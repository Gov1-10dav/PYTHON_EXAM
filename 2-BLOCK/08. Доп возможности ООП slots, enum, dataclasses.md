# II.8 Дополнительные возможности ООП в Python: `__slots__`, `enum`, `dataclasses`

## `__slots__`

### Что это?

Специальная переменная класса, которая **ограничивает набор допустимых атрибутов экземпляра** и замещает словарь `__dict__`.

```python
class Point2D:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y

pt = Point2D(1, 2)
pt.x = 10        # OK
pt.z = 30        # AttributeError: 'Point2D' object has no attribute 'z'
pt.__dict__      # AttributeError: нет __dict__
```

### Три эффекта `__slots__`

1. **Ограничение атрибутов** — нельзя добавить атрибуты вне списка.
2. **Меньше памяти** — вместо словаря `__dict__` (занимает ~88 байт) хранится только пространство имён класса.
3. **Ускорение доступа** к атрибутам — не нужен поиск по хэш-таблице.

```python
# Сравнение памяти
pt = Point(1, 2)       # без слотов
pt2 = Point2D(10, 20)  # со слотами

print(pt.__sizeof__(), pt.__dict__.__sizeof__())  # 32, 88
print(pt2.__sizeof__())                            # 32
```

### Особенности наследования

- `__slots__` **не наследуются автоматически** — дочерний класс без своих слотов снова получает `__dict__`.
- Правильный способ: указывать только **новые** слоты в дочернем классе.

```python
class Point3D(Point2D):
    __slots__ = ('z',)     # только новый атрибут, не дублировать x, y

    def __init__(self, x, y, z):
        super().__init__(x, y)
        self.z = z
```

- Множественное наследование от нескольких классов со слотами запрещено (`TypeError`).
- Чтобы сохранить динамические атрибуты при слотах: добавить `'__dict__'` в кортеж.

### `__slots__` + `@property`

Слоты не мешают свойствам (`@property`) — имена свойств не обязаны быть в `__slots__`, так как это методы, а не атрибуты.

## `Enum`

### Что это?

`Enum` — класс из модуля `enum`, позволяющий создавать **перечисления**: именованные константы с уникальными значениями.

```python
from enum import Enum

class Weekdays(Enum):
    MONDAY    = 1
    TUESDAY   = 2
    WEDNESDAY = 3
    THURSDAY  = 4
    FRIDAY    = 5
    SATURDAY  = 6
    SUNDAY    = 7
```

### Доступ к членам

```python
Weekdays.MONDAY           # Weekdays.MONDAY
Weekdays['MONDAY']        # по имени
Weekdays(1)               # по значению
Weekdays.MONDAY.name      # 'MONDAY'
Weekdays.MONDAY.value     # 1
type(Weekdays.MONDAY)     # <enum 'Weekdays'>
```

### Итерирование и хэширование

```python
for day in Weekdays:
    print(day)            # Weekdays.MONDAY, ...

d = {Weekdays.MONDAY: 'Понедельник'}  # можно использовать в dict/set
```

### Практический пример

```python
class Theme(Enum):
    LIGHT = "light"
    DARK  = "dark"

def set_theme(theme: Theme):
    if theme == Theme.DARK:
        ...  # логика тёмной темы
```

### Преимущества

- Читаемость: именованные константы вместо "магических" строк/чисел.
- Ограничение допустимых значений переменных.
- Возможность хэширования (использование в словарях и множествах).
- Автодокументирование кода.

### Недостатки

- Каждый элемент может иметь только одно значение.
- Значения нельзя менять во время выполнения (immutable).
- Большие перечисления сложнее хранить и обрабатывать.

## `dataclasses`

### Что это?

Декоратор `@dataclass` (доступен с Python 3.7) **автоматически генерирует** методы `__init__`, `__repr__`, `__eq__` и другие на основе полей класса с аннотациями типов.

```python
from dataclasses import dataclass

@dataclass
class Person:
    name: str
    age: int
    address: str = "Unknown"  # значение по умолчанию
```

Эквивалент без `@dataclass`:
```python
class Person:
    def __init__(self, name: str, age: int, address: str = "Unknown"):
        self.name = name
        self.age = age
        self.address = address

    def __repr__(self):
        return f"Person(name={self.name!r}, age={self.age!r}, ...)"

    def __eq__(self, other):
        ...
```

### Создание и использование

```python
p = Person(name="John", age=30)
print(p)  # Person(name='John', age=30, address='Unknown')

# Поддержка методов
@dataclass
class Car:
    make: str
    model: str
    year: int = None

    def age(self):
        return 2025 - self.year if self.year else None
```

### Наследование

```python
@dataclass
class Employee(Person):
    job_title: str
    salary: float
```

### Параметры декоратора

| Параметр | По умолчанию | Назначение |
|----------|-------------|-----------|
| `init=True` | True | Генерировать `__init__` |
| `repr=True` | True | Генерировать `__repr__` |
| `eq=True` | True | Генерировать `__eq__` |
| `order=False` | False | Генерировать `__lt__`, `__gt__`, ... |
| `frozen=False` | False | Запретить изменение полей после создания |

```python
@dataclass(frozen=True)
class User:
    username: str
    password: str

u = User("admin", "secret")
u.username = "hacker"  # FrozenInstanceError
```

### Настройка полей через `field()`

```python
from dataclasses import dataclass, field

@dataclass
class MyClass:
    field1: str
    field2: int = field(default=0)
    field3: str = field(init=False)              # не в __init__
    field4: list = field(default_factory=list)   # изменяемые умолчания
```

### Преимущества

- Краткость: не нужно вручную писать `__init__`, `__repr__`, `__eq__`.
- Встроенные методы сравнения (при `order=True`).
- Поддержка типизации и значений по умолчанию.
- Совместимость с наследованием.

### Ограничения

- Все поля должны быть аннотированы типами.
- Нельзя легко добавить `@property` с логикой.
- `frozen=True` несовместим с мутабельными полями без `field(default_factory=...)`.
- Не заменяет полноценные классы со сложным поведением.

## Дескрипторы (дополнение к билету II.6)

**Дескриптор** — объект, у которого определён хотя бы один из методов: `__get__`, `__set__`, `__delete__`. Позволяет переиспользовать логику геттеров/сеттеров между несколькими атрибутами.

```python
class CoordValue:
    def __get__(self, instance, owner):
        return self.__value

    def __set__(self, instance, value):
        self.__value = float(value)

    def __delete__(self, obj):
        del self.__value

class Point:
    x = CoordValue()
    y = CoordValue()
```
