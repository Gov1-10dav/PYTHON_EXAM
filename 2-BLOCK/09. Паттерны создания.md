# 9. Паттерны проектирования создания (Creational Patterns)

Паттерны создания решают задачи **создания объектов**, скрывая детали инстанцирования и делая код независимым от конкретных классов.

Изученные паттерны (жёлтая разметка): **Фабричный метод**, **Абстрактная фабрика**, **Строитель**, **Прототип**, **Одиночка**.

---

## 1. Фабричный метод (Factory Method)

**Назначение:** Определяет интерфейс для создания объекта, но позволяет подклассам решать, какой класс инстанцировать.

**Когда использовать:**
- Заранее неизвестно, объект какого класса нужно создать.
- Подклассы должны иметь возможность подменять объекты.

```python
from abc import ABC, abstractmethod

class Creator(ABC):
    @abstractmethod
    def factory_method(self):      # возвращает Product
        pass

    def some_operation(self) -> str:
        product = self.factory_method()   # не знает конкретный тип
        return f"Creator worked with {product.operation()}"

class ConcreteCreator1(Creator):
    def factory_method(self):
        return ConcreteProduct1()

class ConcreteCreator2(Creator):
    def factory_method(self):
        return ConcreteProduct2()

class Product(ABC):
    @abstractmethod
    def operation(self) -> str:
        pass

class ConcreteProduct1(Product):
    def operation(self): return "Result of Product1"

class ConcreteProduct2(Product):
    def operation(self): return "Result of Product2"

# Клиентский код работает через базовый интерфейс Creator
def client_code(creator: Creator):
    print(creator.some_operation())

client_code(ConcreteCreator1())
client_code(ConcreteCreator2())
```

**Структура:** `Creator` → `factory_method()` → `Product`

## 2. Абстрактная фабрика (Abstract Factory)

**Назначение:** Предоставляет интерфейс для создания **семейств** взаимосвязанных объектов без указания их конкретных классов.

**Ключевое отличие от Фабричного метода:** создаёт не один, а несколько типов объектов, которые гарантированно совместимы.

```python
from abc import ABC, abstractmethod

class AbstractFactory(ABC):
    @abstractmethod
    def create_product_a(self) -> AbstractProductA: pass

    @abstractmethod
    def create_product_b(self) -> AbstractProductB: pass

class ConcreteFactory1(AbstractFactory):
    def create_product_a(self): return ConcreteProductA1()
    def create_product_b(self): return ConcreteProductB1()

class ConcreteFactory2(AbstractFactory):
    def create_product_a(self): return ConcreteProductA2()
    def create_product_b(self): return ConcreteProductB2()

# Продукты одного семейства могут взаимодействовать
class ConcreteProductB1(AbstractProductB):
    def another_useful_function_b(self, collaborator: AbstractProductA):
        result = collaborator.useful_function_a()
        return f"B1 collaborating with ({result})"

def client_code(factory: AbstractFactory):
    product_a = factory.create_product_a()
    product_b = factory.create_product_b()
    print(product_b.another_useful_function_b(product_a))

client_code(ConcreteFactory1())
```

**Когда использовать:** Систему нужно независить от создания конкретных продуктов; нужно гарантировать, что продукты одного семейства используются вместе.

## 3. Строитель (Builder)

**Назначение:** Разделяет построение сложного объекта от его представления, позволяя создавать разные представления одного и того же процесса строительства.

```python
from abc import ABC, abstractmethod

class Builder(ABC):
    @abstractmethod
    def produce_part_a(self): pass
    @abstractmethod
    def produce_part_b(self): pass
    @abstractmethod
    def produce_part_c(self): pass

class ConcreteBuilder1(Builder):
    def __init__(self):
        self.reset()

    def reset(self):
        self._product = Product1()

    def produce_part_a(self):
        self._product.add("PartA1")

    def produce_part_b(self):
        self._product.add("PartB1")

    def produce_part_c(self):
        self._product.add("PartC1")

    def get_product(self) -> "Product1":
        product = self._product
        self.reset()
        return product

class Product1:
    def __init__(self):
        self.parts = []

    def add(self, part):
        self.parts.append(part)

    def list_parts(self):
        print(f"Product parts: {', '.join(self.parts)}")

# Директор управляет порядком шагов
class Director:
    def __init__(self):
        self._builder = None

    @property
    def builder(self) -> Builder:
        return self._builder

    @builder.setter
    def builder(self, builder: Builder):
        self._builder = builder

    def build_minimal_viable_product(self):
        self.builder.produce_part_a()

    def build_full_featured_product(self):
        self.builder.produce_part_a()
        self.builder.produce_part_b()
        self.builder.produce_part_c()

# Использование
director = Director()
builder = ConcreteBuilder1()
director.builder = builder

director.build_minimal_viable_product()
builder.get_product().list_parts()   # Product parts: PartA1

director.build_full_featured_product()
builder.get_product().list_parts()   # Product parts: PartA1, PartB1, PartC1
```

## 4. Прототип (Prototype)

**Назначение:** Позволяет копировать объекты без привязки к их конкретным классам.

**Когда использовать:** Когда создание объекта с нуля дорого, а можно скопировать уже готовый.

```python
import copy

class SelfReferencingEntity:
    def __init__(self):
        self.parent = None

    def set_parent(self, parent):
        self.parent = parent

class SomeComponent:
    def __init__(self, some_int, some_list_of_objects, some_circular_ref):
        self.some_int = some_int
        self.some_list_of_objects = some_list_of_objects
        self.some_circular_ref = some_circular_ref

    def __copy__(self):
        # Поверхностная копия
        some_list_of_objects = copy.copy(self.some_list_of_objects)
        some_circular_ref = copy.copy(self.some_circular_ref)
        new = self.__class__(
            self.some_int, some_list_of_objects, some_circular_ref
        )
        new.__dict__.update(self.__dict__)
        return new

    def __deepcopy__(self, memo={}):
        # Глубокая копия
        some_list_of_objects = copy.deepcopy(self.some_list_of_objects, memo)
        some_circular_ref = copy.deepcopy(self.some_circular_ref, memo)
        new = self.__class__(
            self.some_int, some_list_of_objects, some_circular_ref
        )
        new.__dict__ = copy.deepcopy(self.__dict__, memo)
        return new

list_of_objects = [1, {1, 2, 3}, [1, 2, 3]]
circular_ref = SelfReferencingEntity()
component = SomeComponent(23, list_of_objects, circular_ref)
circular_ref.set_parent(component)

shallow_copy = copy.copy(component)
deep_copy = copy.deepcopy(component)
```

## 5. Одиночка (Singleton)

**Назначение:** Гарантирует, что класс имеет только **один экземпляр**, и предоставляет к нему глобальную точку доступа.

```python
# Реализация через метакласс
class SingletonMeta(type):
    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            instance = super().__call__(*args, **kwargs)
            cls._instances[cls] = instance
        return cls._instances[cls]

class Singleton(metaclass=SingletonMeta):
    def some_business_logic(self):
        pass

s1 = Singleton()
s2 = Singleton()
print(id(s1) == id(s2))  # True — один и тот же объект
```

**Потокобезопасная версия** (с `Lock`):

```python
from threading import Lock

class SingletonMeta(type):
    _instances = {}
    _lock: Lock = Lock()

    def __call__(cls, *args, **kwargs):
        with cls._lock:             # блокировка перед проверкой
            if cls not in cls._instances:
                instance = super().__call__(*args, **kwargs)
                cls._instances[cls] = instance
        return cls._instances[cls]
```

**Когда использовать:** Конфигурация приложения, логгеры, пулы соединений, кэши.

## Сравнение паттернов создания

| Паттерн | Создаёт | Ключевая идея |
|---------|---------|--------------|
| Фабричный метод | 1 продукт | Подкласс решает, что создать |
| Абстрактная фабрика | Семейство продуктов | Совместимые объекты одной вариации |
| Строитель | Сложный объект пошагово | Директор + Builder разделены |
| Прототип | Копию объекта | `copy.copy` / `copy.deepcopy` |
| Одиночка | Ровно 1 экземпляр | `_instances` + метакласс |
