### **1. Характеристика ООП: классы, экземпляры, методы**
---

#### **Объектно-ориентированное программирование (ООП)** — это парадигма, в которой программа представляется как совокупность взаимодействующих **объектов**.

---
#### 1. Класс (Class)

**Класс** — это шаблон, чертеж или описание того, как должен быть устроен объект. Сам по себе класс ничего не делает, он просто описывает структуру.
Класс описывает, какие **атрибуты** (свойства/данные) и **методы** (действия/функции) будут у объекта.

`# Создание класса (имена классов в Python пишутся с Заглавной буквы - CamelCase)`
`class Car:`
    `pass  # pass означает, что класс пока пустой`

**Пример:** чертеж автомобиля. На чертеже написано, что у машины должны быть колеса, двигатель, цвет. Но по самому чертежу нельзя поехать.

---
#### 2. Экземпляр класса (Instance / Объект)

**Экземпляр** (или просто **объект**) — это конкретная реализация класса, созданная по его чертежу. У каждого экземпляра есть свое собственное состояние (свой набор данных).
Чтобы создать экземпляр, нужно вызвать класс как функцию.

`Создание экземпляров (объектов)`
`car1 = Car()  # Первый экземпляр`
`car2 = Car()  # Второй экземпляр`
`car1 и car2 — это два разных объекта в памяти,хотя они созданы по одному классу.`

**Пример:** конкретный красный автомобиль «Toyota», который только что сошел с конвейера по чертежу `Car`. Завод может сделать миллион экземпляров по одному чертежу.

---

#### 3. Атрибуты (Переменные класса и экземпляра)

*Атрибуты могут быть двух типов: атрибуты класса и атрибуты экземпляра.

- **Атрибуты класса (общие):** 
	Определяются внутри класса, но вне методов. Они **общие для всех экземпляров** (например, количество созданных автомобилей).

- **Атрибуты экземпляра (уникальные):** 
	Определяются внутри методов (обычно в конструкторе) с использованием параметра `self`. Они **уникальны для каждого объекта** (например, цвет конкретного автомобиля).

`class Car:`
    `wheels = 4             # 1. Атрибут класса (общий для всех)`
    
    `def __init__(self, color):`
        `self.color = color   # 2. Атрибут экземпляра (уникальный)`
        
`car1 = Car("red")`
`car2 = Car("blue")`

---
#### 4. Методы (Methods)

**Метод** — это обычная функция, которая определена внутри класса и описывает поведение его экземпляров. Методы говорят объекту, _что он умеет делать_.

**В Python методы делятся на 3 основных вида:**

***Методы экземпляра (Обычные)**
- **Первый параметр:** `self` (ссылка на конкретный объект).
- **Суть:** имеют доступ к атрибутам _конкретного объекта_ (через `self.атрибут`). Самый частый и основной тип методов. 

***Методы класса (Class methods)**
- **Декоратор:** `@classmethod`.
- **Первый параметр:** `cls` (ссылка на сам класс).
- **Суть:** привязаны к самому классу, а не к его отдельным копиям. Они могут менять состояние класса в целом, но не имеют доступа к конкретным объектам.

***Статические методы (Static methods)***
- **Декоратор:** `@staticmethod`.
- **Параметры:** Не требуют ни `self`, ни `cls`.
- **Суть:** Это просто обычные функции, которые логически относятся к классу, но никак не взаимодействуют с его данными.

### Кратко:

| Термин        | Простыми словами                                  | Пример в коде         | Аналогия                    |
| ------------- | ------------------------------------------------- | --------------------- | --------------------------- |
| **Класс**     | Шаблон, чертеж. Описывает структуру.              | `class Dog:`          | Собака                      |
| **Экземпляр** | Конкретный объект, созданный по шаблону.          | `my_dog = Dog()`      | Моя собака                  |
| **Атрибут**   | Переменная внутри объекта (его данные/состояние). | `self.name = "Шарик"` | Кличка собаки или ее порода |
| **Метод**     | Функция внутри объекта (его поведение).           | `def bark(self):`     | Лай собаки                  |
