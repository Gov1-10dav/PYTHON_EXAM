## Сущность

**Магические методы** (dunder-методы, от *double underscore*) — специальные методы Python, обрамлённые двойными нижними подчёркиваниями (`__name__`). Они позволяют классам «имитировать» встроенные типы и перегружать стандартные операторы, делая пользовательские объекты интуитивно понятными.

```python
class Money:
    def __init__(self, amount):
        self.amount = amountC:\Users\mrsor\Downloads\7. Магические методы

    def __add__(self, other):
        return Money(self.amount + other.amount)

    def __repr__(self):
        return f"Money({self.amount})"

a = Money(100)
b = Money(50)
print(a + b)  # Money(150)
```

## Классификация

### 1. Конструирование и инициализация

| Метод | Назначение |
|-------|-----------|
| `__new__(cls, ...)` | Создаёт экземпляр (вызывается до `__init__`). Нужен при наследовании от иммутабельных типов (str, tuple) |
| `__init__(self, ...)` | Инициализирует атрибуты объекта |
| `__del__(self)` | Деструктор; вызывается при попадании объекта в сборщик мусора (не гарантирован!) |

```python
class FileObject:
    def __init__(self, path):
        self.file = open(path, 'r+')

    def __del__(self):
        self.file.close()
        del self.file
```

---

### 2. Методы сравнения

| Метод | Оператор |
|-------|----------|
| `__eq__(self, other)` | `==` |
| `__ne__(self, other)` | `!=` |
| `__lt__(self, other)` | `<` |
| `__gt__(self, other)` | `>` |
| `__le__(self, other)` | `<=` |
| `__ge__(self, other)` | `>=` |
| `__cmp__(self, other)` | универсальный (Python 2) |

Совет: не нужно определять все методы — достаточно `__eq__` и одного из `__lt__/__gt__`, а остальное дополнит `@total_ordering` из `functools`.

```python
class Word(str):
    def __gt__(self, other): return len(self) > len(other)
    def __lt__(self, other): return len(self) < len(other)
```

---

### 3. Числовые / арифметические методы

**Унарные:** `__pos__`, `__neg__`, `__abs__`, `__invert__`, `__round__`, `__floor__`, `__ceil__`, `__trunc__`

**Бинарные (обычные):** `__add__`, `__sub__`, `__mul__`, `__div__`, `__truediv__`, `__floordiv__`, `__mod__`, `__pow__`, `__lshift__`, `__rshift__`, `__and__`, `__or__`, `__xor__`

**Отражённые (`r`-варианты):** `__radd__`, `__rsub__`, ... — вызываются, если левый операнд не имеет нужного метода.

**Составное присваивание:** `__iadd__`, `__isub__`, `__imul__`, ... (для `+=`, `-=`, `*=` и т.д.)

**Преобразование типов:** `__int__`, `__float__`, `__complex__`, `__bool__`, `__str__`, `__repr__`, `__bytes__`

---

### 4. Представление объекта

| Метод | Когда вызывается |
|-------|-----------------|
| `__str__(self)` | `str(obj)`, `print(obj)` — для пользователя |
| `__repr__(self)` | `repr(obj)`, отладка — должен быть однозначным |
| `__format__(self, spec)` | `format(obj, spec)`, f-строки |

---

### 5. Итерирование

| Метод | Назначение |
|-------|-----------|
| `__iter__(self)` | Возвращает итератор (делает объект итерируемым) |
| `__next__(self)` | Возвращает следующий элемент (или `StopIteration`) |
| `__len__(self)` | Длина объекта, `len(obj)` |
| `__contains__(self, item)` | Оператор `in` |

---

### 6. Доступ к атрибутам

| Метод | Назначение |
|-------|-----------|
| `__getattr__(self, name)` | Вызывается, когда атрибут **не найден** обычным способом |
| `__setattr__(self, name, value)` | При любом присваивании атрибута |
| `__delattr__(self, name)` | `del obj.attr` |
| `__getattribute__(self, name)` | Вызывается при **каждом** обращении к атрибуту |

---

### 7. Контейнерные (subscription) методы

| Метод | Оператор |
|-------|----------|
| `__getitem__(self, key)` | `obj[key]` |
| `__setitem__(self, key, value)` | `obj[key] = value` |
| `__delitem__(self, key)` | `del obj[key]` |

---

### 8. Callable

```python
class Multiplier:
    def __init__(self, factor):
        self.factor = factor

    def __call__(self, x):
        return x * self.factor

double = Multiplier(2)
print(double(5))  # 10
```

---

### 9. Контекстный менеджер

| Метод | Назначение |
|-------|-----------|
| `__enter__(self)` | Выполняется при входе в блок `with` |
| `__exit__(self, exc_type, exc_val, exc_tb)` | Выполняется при выходе (в т.ч. при исключении) |

## Достоинства

- **Pythonic-код**: объекты ведут себя как встроенные типы, код читается естественно.
- **Перегрузка операторов**: `+`, `-`, `*`, `[]`, `in` и другие работают с пользовательскими классами.
- **Интеграция со стандартной библиотекой**: `len()`, `sorted()`, `print()`, `with` — всё автоматически поддерживается.
- **Гибкость**: можно определить только нужные методы.

## Ограничения

- Не рекомендуется злоупотреблять `__getattribute__` — легко сломать всё поведение класса.
- `__del__` ненадёжен: не гарантируется момент вызова, особенно при циклических ссылках.
- Перегрузка операторов может запутать код, если семантика неочевидна.
- `__new__` нужен редко и требует аккуратности (особенно при иммутабельных базовых классах).
