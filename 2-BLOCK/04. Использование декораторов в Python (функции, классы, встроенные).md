### 4. Использование декораторов в Python (функции, классы, встроенные)
---
#### **Декоратор** — это функция или объект, который принимает другую функцию (или класс), изменяет её поведение и возвращает новый вызываемый объект. При этом исходная функция не изменяется.

**Общий вид:**
```
def decorator(func):
    def wrapper():
        print("До")
        func()
        print("После")
    return wrapper
```
**Использование:**
```
@decorator
def hello():
    print("Hello")
```
**Эквивалентно:**
```
hello = decorator(hello)
```
---
#### **Декораторы функций**
Декораторы функций - основной способ использования.
Пример:
```
@uppercase
def greet():
    return "Hello"
```
После декорирования функция возвращает `"HELLO"`.
Чтобы декоратор работал с любой функцией (с любым числом аргументов), используют `*args` и `**kwargs`:
```
def decorator(func):
    def wrapper(*args, **kwargs):
        print(f"Вызов функции {func.__name__}")
        result = func(*args, **kwargs)
        print(f"Функция выполнена, результат: {result}")
        return result
    return wrapper

@decorator
def add(a, b):
    return a + b

add(3, 5)
# Вызов функции add
# Функция выполнена, результат: 8
```
---
#### **Декоратор, изменяющий поведение**
Чтобы что-то делать до/после вызова функции, внутри декоратора создаётся функция-обёртка `wrapper`:
```
def uppercase(func):
    def wrapper():
        result = func()
        return result.upper()  # изменяем результат
    return wrapper

@uppercase
def greet():
    return "Hello, world!"

print(greet())  # HELLO, WORLD!
```
Что происходит: `greet` теперь ссылается на `wrapper`. При вызове `greet()` выполняется `wrapper()`, который внутри себя вызывает оригинальную функцию.

---
#### **Несколько декораторов (стек)**
К одной функции можно применить несколько декораторов. Порядок применения — **снизу вверх**, а порядок выполнения обёрток — **сверху вниз**:
```
def bold(func):
    def wrapper():
        return "<b>" + func() + "</b>"
    return wrapper

def italic(func):
    def wrapper():
        return "<i>" + func() + "</i>"
    return wrapper

@bold      # применяется вторым
@italic    # применяется первым
def greet():
    return "Hello"

print(greet())  # <b><i>Hello</i></b>
```

#### **Декоратор с параметрами**
Если нужно передать параметр в сам декоратор, добавляется ещё один уровень вложенности — фабрика декораторов:
```
def repeat(times):          # это фабрика — принимает аргумент
    def decorator(func):    # это настоящий декоратор
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)  # передали аргумент, а именно количество повторов
def say_hi():
    print("Hi!")

say_hi()
# Hi!
# Hi!
# Hi!
```
Что происходит: `@repeat(3)` сначала вызывает `repeat(3)`, который возвращает `decorator`. Затем `decorator` применяется к `say_hi`.

#### **Декораторы-классы**
Класс становится декоратором, если реализует метод `__call__`. Это позволяет хранить состояние между вызовами:
```
class CountCalls:
    def __init__(self, func):
        self.func = func
        self.count = 0  # состояние — сколько раз вызвали

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Вызов #{self.count}")
        return self.func(*args, **kwargs)

@CountCalls
def say_hello():
    print("Hello!")

say_hello()  # Вызов #1 / Hello!
say_hello()  # Вызов #2 / Hello!
say_hello()  # Вызов #3 / Hello!
```
**Декоратор-класс с параметром** — `__init__` принимает параметры, `__call__` принимает функцию и возвращает обёртку:
```
class Repeat:
    def __init__(self, times):  # параметры декоратора
        self.times = times

    def __call__(self, func):   # здесь func — декорируемая функция
        def wrapper(*args, **kwargs):
            for _ in range(self.times):
                func(*args, **kwargs)
        return wrapper

@Repeat(times=3)
def greet():
    print("Hello!")

greet()  # Hello! / Hello! / Hello!
```

#### **Встроенные декораторы Python**
- `@property`
	Превращает методы в управляемые атрибуты.

- `@classmethod`
	Метод привязан к классу, а не к экземпляру. Получает класс (`cls`) как первый аргумент. Используется чаще всего как **альтернативный конструктор**.
```
		
	class Circle:
	def __init__(self, radius):
	    self.radius = radius
	
	@classmethod
	def from_diameter(cls, diameter):  # альтернативный конструктор
	    return cls(diameter / 2)
	
	c = Circle.from_diameter(10)
	print(c.radius)  # 5.0
		
```

- `@staticmethod`
	Метод не привязан ни к классу, ни к экземпляру — это обычная функция, просто логически размещённая внутри класса.
```
	class Student:
    @staticmethod
    def is_valid_age(age):
        return 5 <= age <= 100

	print(Student.is_valid_age(25))   # True
	print(Student.is_valid_age(200))  # False
```

- `@lru_cache`
	Кэширует результаты функции. При повторном вызове с теми же аргументами возвращает сохранённый результат, не вычисляя заново.

- `@dataclass`
```
	Автоматически генерирует `__init__`, `__repr__`, `__eq__` и другие методы:
	from dataclasses import dataclass

	@dataclass
	class Point:
	    x: float
	    y: float

	p = Point(1.0, 2.0)
	print(p)       # Point(x=1.0, y=2.0) — __repr__ сгенерирован автоматически
	print(p.x)     # 1.0
```
