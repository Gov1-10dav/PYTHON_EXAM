# 11. Обработка исключений в Python

Исключение — это ошибка, которая возникает во время выполнения программы.

Если исключение не обработать, программа остановится с ошибкой.

Пример ошибки:

```python
x = int("abc")
```

Результат:

```python
ValueError
```

Ошибка возникает потому, что строку `"abc"` нельзя превратить в число.

## Зачем нужна обработка исключений

Обработка исключений нужна, чтобы программа не падала при ошибке, а могла нормально продолжить работу или вывести понятное сообщение.

Например, пользователь может ввести не число, файл может не открыться, интернет может пропасть, ключа в словаре может не быть.

## Конструкция `try except`

Основная конструкция для обработки ошибок:

```python
try:
    # код, где может быть ошибка
except:
    # код, который выполнится при ошибке
```

Пример:

```python
try:
    x = int(input("Введите число: "))
    print(x)
except:
    print("Ошибка: нужно ввести число")
```

Если пользователь введет текст вместо числа, программа не упадет.

## Обработка конкретного исключения

Лучше указывать конкретный тип ошибки.

```python
try:
    x = int(input("Введите число: "))
    print(10 / x)
except ValueError:
    print("Ошибка: нужно ввести число")
except ZeroDivisionError:
    print("Ошибка: делить на ноль нельзя")
```

Здесь обрабатываются две разные ошибки:

- `ValueError` — если введено не число;
- `ZeroDivisionError` — если пользователь ввел `0`.

## Блок `else`

`else` выполняется, если ошибок не было.

```python
try:
    x = int(input("Введите число: "))
except ValueError:
    print("Ошибка: нужно ввести число")
else:
    print("Вы ввели:", x)
```

Если ошибка возникла, `else` не выполнится.

## Блок `finally`

`finally` выполняется всегда: была ошибка или нет.

```python
try:
    file = open("data.txt", "r", encoding="utf-8")
    text = file.read()
except FileNotFoundError:
    print("Файл не найден")
finally:
    print("Завершение работы")
```

`finally` удобно использовать, когда нужно обязательно выполнить какое-то действие, например закрыть файл или соединение.

## Полная структура `try except else finally`

```python
try:
    # код, где может возникнуть ошибка
except ValueError:
    # обработка конкретной ошибки
else:
    # выполнится, если ошибки не было
finally:
    # выполнится всегда
```

## Частые виды исключений

| Исключение | Когда возникает |
|---|---|
| `ValueError` | неверное значение, например `int("abc")` |
| `ZeroDivisionError` | деление на ноль |
| `TypeError` | операция с неправильным типом данных |
| `IndexError` | обращение к несуществующему индексу списка |
| `KeyError` | обращение к несуществующему ключу словаря |
| `FileNotFoundError` | файл не найден |
| `NameError` | переменная не определена |

## Пример с `IndexError`

```python
numbers = [10, 20, 30]

try:
    print(numbers[5])
except IndexError:
    print("Такого индекса нет")
```

Результат:

```python
Такого индекса нет
```

## Пример с `KeyError`

```python
student = {"name": "Иван", "age": 18}

try:
    print(student["group"])
except KeyError:
    print("Такого ключа нет")
```

Результат:

```python
Такого ключа нет
```

## Пример с несколькими исключениями

```python
try:
    a = int(input("Введите первое число: "))
    b = int(input("Введите второе число: "))
    result = a / b
    print(result)
except ValueError:
    print("Нужно вводить только числа")
except ZeroDivisionError:
    print("На ноль делить нельзя")
```

## Как получить текст ошибки

Можно сохранить ошибку в переменную.

```python
try:
    x = int("abc")
except ValueError as error:
    print("Произошла ошибка:", error)
```

Результат:

```python
Произошла ошибка: invalid literal for int() with base 10: 'abc'
```

## Генерация своего исключения: `raise`

`raise` используется, чтобы самостоятельно вызвать ошибку.

```python
age = -5

if age < 0:
    raise ValueError("Возраст не может быть отрицательным")
```

## Главное

Обработка исключений нужна, чтобы программа не завершалась аварийно при ошибках.

Самое важное:

- `try` — код, где может быть ошибка;
- `except` — обработка ошибки;
- `else` — выполняется, если ошибки не было;
- `finally` — выполняется всегда;
- лучше обрабатывать конкретные исключения, а не писать просто `except`.
