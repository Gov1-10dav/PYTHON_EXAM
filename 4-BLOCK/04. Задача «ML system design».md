### 4. Задача «ML system design»
---
#### **ML System Design** — проектирование системы машинного обучения от постановки задачи до внедрения модели в эксплуатацию.

---
#### **Основные этапы**

| Этап                                     | Особенность                                                                                                                                                                                                                                                                                                                                                                  | Этап                                 | Особенность                                                                                                                                                                                                                                                                                        |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. Постановка задачи**                 | Определить:<br>- цель проекта;<br>- входные данные;<br>- ожидаемый результат;<br>- ограничения.<br><br>Также определяют, какая это задача: **классификация, регрессия или кластеризация**.                                                                                                                                                                                   | **6. Обучение модели**               | На обучающей выборке модель ищет закономерности между признаками и правильными ответами.<br><br>Во время обучения она постепенно изменяет свои параметры, чтобы уменьшить ошибку предсказаний.                                                                                                     |
| **2. Сбор данных**                       | На этом этапе собирают данные, на которых будет обучаться модель.<br><br>Источниками могут быть:<br>- базы данных;<br>- CSV- и Excel-файлы;<br>- API;<br>- датчики;<br>- логи приложений;<br>- веб-сайты.<br><br>**Чем больше и качественнее данные, тем лучше обычно работает модель.**                                                                                     | **7. Оценка качества модели**        | После обучения проверяют, насколько хорошо модель работает на новых данных.<br><br>Используют различные метрики.<br><br>Если качество недостаточное, возвращаются к предыдущим этапам: изменяют признаки, параметры модели или выбирают другой алгоритм.                                           |
| **3. Подготовка данных (предобработка)** | Сырые данные редко подходят для обучения, поэтому их необходимо обработать.<br><br>Включает:<br>- удаление пропусков;<br>- обработку выбросов;<br>- кодирование категорий;<br>- масштабирование;<br>- создание новых признаков.                                                                                                                                              | **8. Внедрение модели (Deployment)** | Если качество модели устраивает, её внедряют в реальную систему.<br><br>Например, модель может работать:<br>- в веб-приложении;<br>- в мобильном приложении;<br>- как REST API;<br>- как отдельный сервис.<br><br>После этого пользователи начинают получать предсказания модели.                  |
| **4. Разделение данных**                 | Данные делят на несколько частей:<br><br>- **Train (обучающая выборка)** — используется для обучения модели;<br>- **Validation (валидационная выборка)** — используется для подбора параметров модели;<br>- **Test (тестовая выборка)** — используется для окончательной проверки качества модели.<br><br>Часто используют разделение **70% / 15% / 15%** или **80% / 20%**. | **9. Мониторинг и обновление**       | После внедрения работу модели необходимо постоянно контролировать.<br><br>Следят за:<br>- качеством предсказаний;<br>- скоростью работы;<br>- изменением входных данных (**Data Drift**).<br><br>Если данные со временем изменились и качество ухудшилось, модель **переобучают на новых данных**. |
| **5. Выбор модели**                      | Выбирают алгоритм, который лучше подходит для задачи.<br><br>Например:<br><br>- **Linear Regression** — для прогнозирования чисел;<br>- **Logistic Regression** — для классификации;<br>- **Decision Tree**;<br>- **Random Forest**;<br>- **Gradient Boosting**;<br>- **Neural Network**.                                                                                    |                                      |                                                                                                                                                                                                                                                                                                    |

---
#### **Краткая схема**

| Постановка задачи<br>        ↓<br>Сбор данных<br>        ↓<br>Подготовка данных<br>        ↓<br>Разделение на train / validation / test<br>        ↓<br>Выбор модели<br>        ↓<br>Обучение модели<br>        ↓<br>Оценка качества<br>        ↓<br>Внедрение (Deployment)<br>        ↓<br>Мониторинг и переобучение |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
